# Code of Conduct

## Our pledge

We want this project to be useful and welcoming to everyone who shows up to contribute. That means treating each other with basic respect regardless of background, experience level, or which side of any debate someone falls on.

## Standards

Behavior we want:

- Constructive feedback. Be specific about what you mean.
- Patience with people learning the codebase or the problem domain.
- Crediting prior work when it informs your contribution.

Behavior we don't want:

- Personal attacks, slurs, or harassment.
- Sustained off-topic noise in issues or PRs.
- Sharing someone's private information without consent.
- Behavior that would be inappropriate in a professional engineering setting.

## Enforcement

If something feels off, email **swayam.dash@unscript.ai** privately. The maintainer will review and respond within a few days. Possible outcomes range from a private clarification to removal from the project.

We try to apply judgment over rules. The goal is a project people want to keep contributing to.

## Attribution

This is a lightweight code of conduct adapted from common open-source norms (see [Contributor Covenant](https://www.contributor-covenant.org) for a more formal version).
